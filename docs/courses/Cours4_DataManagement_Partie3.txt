Les valeurs manquantes
3 Stratégies pour traiter les valeurs manquantes:
1.

Supprimer toutes les lignes ayant des valeurs manquantes –> A ne pas faire si pas beaucoup de données
- Possibilité de se retrouver avec 0 observation
- Les données ne seront pas évaluées

2.

Supprimer toutes les variables ayant des valeurs manquantes –> A ne pas faire sauf si la colonne est pratiquement
vide
- Perte d’information
- Il peut y avoir une raison expliquant pourquoi une variable possède des valeurs manquantes

3.

Imputer les valeurs manquantes –> Meilleure stratégie composée de 3 techniques principales

Les valeurs manquantes
Techniques d’imputation pour variables quantitatives:


Imputer par la valeur qu’il devrait normalement y avoir si on sait pourquoi une donnée est manquante



Imputer par une valeur impossible mais dans la même échelle de grandeur que la variable



Imputer par une valeur possible (ex: Moyenne | Médiane ou Moyenne | Médiane en fonction d’autres variables
ATTENTION: Cela introduit un biais de sur-représentation de la valeur vis-à-vis de la population

Bonus : les imputations basées sur du machine learning –> Imputation prédite. Ex : régression, arbres de décision.
 Avantage : peut capturer des relations complexes entre variables.
 Inconvénient : souvent calculatoire, parfois opaque (“black box”), et le gain peut être marginal selon le contexte.

Conseil:
Créer une variable indicatrice binaire : 1 si la valeur a été imputée, 0 sinon.
Permet à l’algorithme d’utiliser l’information “manquante” comme un signal supplémentaire.

Les valeurs manquantes
Techniques d’imputation pour variables qualitatives:
1. Si vous savez pourquoi une donnée est manquante: Imputer par la catégorie qui y réfère
2. Imputer par la catégorie la plus fréquente en fonctions des variables (mode)
3. Renommer la catégorie en « Missing »

Les valeurs aberrantes
Une anomalie dans les données, également appelée valeur aberrante (ou ‘outlier’), est une observation
qui diffère sensiblement par rapport au modèle standard.
• Des anomalies dans les données peuvent survenir pour diverses raisons, erreur humaine, arrêt de
système …
 Considérons la série donnée contenant la largeur d'un tableau mesurée par 5 élèves différents.
- La troisième valeur est significativement différente des autres –> outlier

Les valeurs aberrantes
•Pour détecter les outliers, il faut utiliser des méthodes statistiques comme la médiane ou la
moyenne ou avec des visualisations
•Ensuite, il faut supprimer ou remplacer les valeurs aberrantes selon le cas suivant les même
méthodes que pour les valeurs manquantes

Méthode utiles
Plusieurs méthodes utilisées dans des projets de Big Data & ML :
• apply() qui permet d’appliquer une fonction sur l’ensemble d’un dataframe ou une série
• sort_values() qui permet de trier un dataframe dans l’ordre croissant ou décroissant
• corr() qui permet d’afficher la matrice de corrélation entre les différentes variables
• idxmin & idxmax qui retourne l’index de la première occurrence du min / max dans la colonne
choisie
• value_counts qui permet de compter le nombre d’occurrence de chaque valeur d’une colonne et les
retourner dans l’ordre décroissant
• replace qui permet de remplacer une valeur par une autre valeur définie
• unique & nunique qui permet de retourner les valeurs uniques & le nombre des valeurs uniques
• map qui permet d’appliquer une fonction sur chaque élément d’une série
• between qui retourne True or False en fonction de si l’élement de la série se retrouve en les deux
valeurs indiquées dans le between
• sample qui retourne un échantillon aléatoire d'éléments à partir des données du dataframe (utilisée
pour décomposer notre dataframe en base de train & en base de test de façon aléatoire sans passer
par la fonction train_test_split de scikit-learn ).
• nlargest qui permet de sélectionner les n plus grandes valeurs
